This file is the **rights layer** for LetPeopleWorkShop's facilitation practice library. It records,
for every methodology the library draws on, who originated it and under what terms — so the project
stays honest and safe as new methods are added.

> **Not legal advice.** This documents our good-faith attribution and the basis on which we include
> each method. If you redistribute or build a commercial offering on this, confirm the source terms

## Adding a new method — the four gates

Before a new practice goes in (whether added by a human or the `designer`), it must pass all four.
This is what keeps the dual-license valid no matter how commercially protected the source is.

1. **Original expression only.** Write it in *our* words and *our* schema — never copy or
   close-paraphrase the source's text.
2. **No protected instruments reproduced.** Don't reproduce a source's proprietary worksheet, diagram,
   assessment, or exact question sequence. Describe the *process*. **If a method can't be described
   without copying its instrument, it does not go in.**
3. **Attribute the originator** — `source` + `source_url` in frontmatter, and a credit line in the body.
4. **Disclaim affiliation + acknowledge trademark** — add a row here; the standing disclaimer above
   covers endorsement; add a ™/® note for any branded name.

Then record the source in the table above (with its rights status), and set the practice's frontmatter
`rights_status` / `attribution` / `trademark` fields where relevant.
