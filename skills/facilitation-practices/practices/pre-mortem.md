# Pre-mortem

- **Source detail:** Technique by **Gary Klein** ("Performing a Project Premortem," *Harvard Business Review*, Sept. 2007 — based on prospective hindsight); catalogued in *Gamestorming* (Gray, Brown, Macanufo). The positive mirror is `remember-the-future`.
- **Purpose:** Surface risks honestly by imagining the plan has *already failed* and asking why — prospective hindsight gives people permission to voice doubts that optimism and loyalty would otherwise suppress.
- **When to use:** Right after a plan or decision feels settled but **before** you commit to it, when you want real risks (not polite caveats) on the table — especially around a leader's pet plan.
- **Group config:** Silent solo write -> round-robin harvest -> cluster + assign mitigations.
- **Timing:** ~20-45 min (~5 min/participant write + harvest + mitigations). Trim by capping the write and harvesting top risks only.
- **Medium fit:**
  - In-person: stickies on a "post-mortem" wall; round-robin to the board. Excellent.
  - Miro: a "why it failed" frame, then drag into clusters and add mitigations.
  - Video: silent doc write, then round-robin read-aloud; cluster live.
- **Steps:**
  1. With the plan fresh and agreed, set the scene: "Imagine it's [future date] and this has failed — badly. A fiasco."
  2. Each person **silently writes** the reasons it failed, as if it already happened — concrete causes, not vague worries (~5 min).
  3. Round-robin: each reads one reason; build a visible list, no debating yet.
  4. Cluster; pick the most plausible/damaging; assign mitigations or adjust the plan.
- **Facilitator notes:** Framing failure as a **foregone fact** ("it failed — why?") unlocks honesty that "what could go wrong?" never will — that's the whole point, so don't soften it back into a risk-register. Do it after the plan feels settled but before commitment, so there's still time to act. Insist on the silent individual write first (same logic as `1-2-4-all`): it stops the optimistic loudest voice from setting the tone.
