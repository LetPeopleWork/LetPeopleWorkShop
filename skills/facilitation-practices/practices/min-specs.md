# Min Specs

- **Purpose:** Find the minimum set of must-do and must-not-do rules needed to reach a goal, by listing every "must" a group can think of and then ruthlessly eliminating the ones that aren't truly essential.
- **When to use:** When a process, plan, or set of constraints has become bloated and over-specified; to free people to act with maximum creativity inside the smallest possible boundary of rules.
- **Group config:** Small groups (e.g. 4s) generate and test the specs -> whole group reconciles into one shared minimum set.
- **Timing:** ~35-50 min (list all "musts" ~10 min, elimination test ~15-20 min, whole-group reconcile ~10-15 min). Trim to ~25 min with smaller scope and one fast reconcile.
- **Medium fit:**
  - In-person: stickies for every "must" on a wall, then physically pull off the ones that fail the test.
  - Miro: a column of Max Specs, dragging survivors into a Min Specs column as each passes the test.
  - Video: shared doc/board listing all musts, then breakouts run the elimination test, then plenary merge.
- **Steps:**
  1. Clarify the goal/purpose everyone is trying to reach.
  2. List ALL the "musts" — every must-do and must-not-do rule the group believes is required (the Max Specs).
  3. Test each one: "If we violated or ignored this rule, could we still reach the goal?" If yes, eliminate it.
  4. Keep only the specs that fail the test (i.e. truly indispensable) — these are the Min Specs.
  5. Whole group reconciles the surviving specs into one agreed minimum set.
- **Facilitator notes:** The elimination test is the entire engine — most "musts" are habits, preferences, or fears in disguise, and they only reveal themselves when you ask "would we still hit the goal without this?" Do NOT let the group defend a rule emotionally; make them answer the test honestly. Aim to cut the list dramatically — if almost everything survives, the group is testing too softly.
