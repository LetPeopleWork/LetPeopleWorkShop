# The 4 Cs — session design framework (TBR)

- **Source detail:** Sharon Bowman, *Training from the BACK of the Room*. The map for sequencing a
  learning experience so it lands.
- **Purpose:** give a session a brain-friendly **backbone** — connect, deliver, practice, conclude —
  onto which you map concrete grounded structures.
- **When to use:** as the macro-arc for **learning/training-oriented** sessions. For pure
  problem-surfacing or decision sessions, use it *loosely* — not every workshop is a training (see notes).
- **Medium fit:** all.

## The four Cs (in order)
1. **Connections** — learners connect to the topic, to each other, and to what they already know/want
   (this is where **WIIFM** lives). → map: openers like `impromptu-networking`.
2. **Concepts** — the core content, delivered **interactively** (not lecture); learners process it
   actively. → map: `1-2-4-all` and surfacing/teaching structures.
3. **Concrete Practice** — learners actively **apply/practice** the content. → map: practice/application
   structures, scenarios, try-it rounds.
4. **Conclusions** — learners **summarize, evaluate, and commit** to next steps — *learner-led*, not a
   facilitator recap. → map: `what-so-what-now-what`.

## How to use it
Shape the agenda so it moves through the 4Cs; each C holds one or more grounded structures. Always
**open in Connections** and **close in Conclusions**. Keep the `tbr-brain-rules` Six Trumps active within
each C.

## Facilitator notes
The 4Cs is a **learning-design** map; apply judgement. A problem-surfacing session (e.g. cross-team
collaboration) *bends* it — heavy on Connections + "Concepts as surfacing", light on Concrete Practice,
firm Conclusions. Don't force four labelled blocks onto every session. (Worked example: the cross-team
design maps loosely as Connections=`impromptu-networking`, Concepts=`1-2-4-all`/`triz`,
Conclusions=`what-so-what-now-what`.)
