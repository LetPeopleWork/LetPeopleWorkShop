# Practices library

The grounding for every workshop design. Each file is one facilitation **structure** the `designer`
agent can compose into an agenda. The library is intentionally extensible — **add a practice by
adding a markdown file**, no code change.

## How to add a practice
1. Copy `_TEMPLATE.md` to a kebab-case name (e.g. `min-specs.md`).
2. Fill every field. `Medium fit` and `Facilitator notes` matter most — the designer leans on them.
3. That's it. The designer globs `practices/*.md` and will use it next time.

## Sources seeded so far
